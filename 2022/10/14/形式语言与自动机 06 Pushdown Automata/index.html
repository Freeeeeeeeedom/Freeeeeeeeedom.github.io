Pushdown Automata

下推自动机，表达能力和上下文无关语言等价

Definition

  • 下推自动机的语言表达能力和CFG等价
  • 仅有非确定性下推自动机能定义所有的上下文无关语言
  • 确定性PDA可以对解析器进行建模
    • 大部分编程语言都有一个确定性下推自动机

Intuition: PDA

  • 想象一个ϵ−\epsilon-ϵ−NFA 额外控制一个栈
  • 它的移动由以下几点决定
    • 当前的状态
    • 当前的输入字符
    • 当前的栈顶元素

Picture of a PDA

Intuition: PDA

  • Being nondeterministic, the PDA can have a choice of next moves
  • In each choice, the PDA can:
    • 改变状态，并且
    • 替换栈顶元素
      • Zero symbols = “pop”
      • Many symbols = sequence of “pushes”

PDA Formalism

  1. 有穷状态集合 (Q, typically)
  2. 输入字母表 (Σ\SigmaΣ, typically)
  3. 栈字母表 (Γ\GammaΓ, typically)
  4. 转移函数 (δ\deltaδ, typically)
  5. 初始状态(q0q_0q0​, in Q, typically)
  6. 初始字符 (Z0,∈ΓZ_0, \in \GammaZ0​,∈Γ, typically)
  7. 终止状态(F⊆QF\subseteq QF⊆Q, typically)

Conventions

约定

  • a,b,…都是输入字符
  • …,X,Y,Z是栈元素
  • …,w,x,y,z是输入字符
  • α,β,\alpha,\beta,α,β,…是栈字符串

The Transition Function

  • 接受三个参数
    • 当前状态
    • 输入（字母表中的元素或空串）
    • 栈元素
  • δ(q,a,Z)\delta (q,a,Z)δ(q,a,Z)是一组(p,α)(p,\alpha)(p,α)形式的操作
    • p是一个状态，α\alphaα是栈字符串

Actions of the PDA

  • 如果δ(q,a,Z)\delta(q,a,Z)δ(q,a,Z)包括(p,α)(p,\alpha)(p,α),那么PDA的行为可以有（q为当前状态,a为当前input元素， Z为栈顶元素）
    • 将状态修改为p
    • 移除a元素（a可能是空串）
    • 将栈顶元素替换为α\alphaα

Example PDA

构造一个接受0n1n0^n1^n0n1n的PDA

  • q为初始状态，如果仅遇到1，那么一直为q状态
  • p状态表示至少看到一个1，且仅当继续输入1才能继续
  • f为终止状态，表示接受了改字符串

栈元素

  • Z0Z_0Z0​表示初始元素，也代表栈底元素，所以我们知道何时计算了同样数量的0和1
  • X代表标记，用于统计input中已经检测到的0的个数

转移函数

  • δ(q,0,Z0)={(q,XZ0)}\delta(q,0,Z_0) = \{(q,XZ_0)\}δ(q,0,Z0​)={(q,XZ0​)}
  • δ(q,0,X)={q,XX}\delta(q,0,X)=\{q,XX\}δ(q,0,X)={q,XX} 这两个规则可以使得input中的每个0入栈
  • δ(q,1,X)={p,ϵ}\delta(q,1,X) = \{p,\epsilon \}δ(q,1,X)={p,ϵ} 当读到1时，进入p状态，并且将一个X出栈
  • δ(p,1,X)={(p,ϵ)}\delta(p,1,X) = \{(p,\epsilon)\}δ(p,1,X)={(p,ϵ)} 每读取一个1，出栈一个X
  • δ(p,ϵ,Z0)={(f,Z0)}\delta(p,\epsilon,Z_0) = \{(f,Z_0)\}δ(p,ϵ,Z0​)={(f,Z0​)}，到达栈底，接受该元素

Instantaneous Descriptions

即时描述

我们可以将图形化表达变成转换成即时描述（ID）

一个ID是一个三元组(q,a,α)(q,a,\alpha)(q,a,α)

  • q是当前状态
  • w是剩余输入
  • α\alphaα是栈内容，最左边

The “Goes-To” Relation

  • 如果ID I通过一次转移函数变成ID J，我们写作 I⊢JI \vdash JI⊢J
  • 一般来说，(q,aw,Xα)⊢(p,w,βα)(q,aw,X\alpha) \vdash (p,w,\beta\alpha)(q,aw,Xα)⊢(p,w,βα) （如果δ(q,a,X)\delta(q,a,X)δ(q,a,X) contains (p,β)(p,\beta)(p,β)）
  • 拓展 ⊢∗\vdash ^*⊢∗

Deterministic PDA’ s

  • To be deterministic, there must be at most one choce of move for any state q, input symbol a, and stack symbol X
  • In addition, there must not be a choice between using input ∈\in∈ or real input
    • Formally, δ(q,a,X)\delta(q,a,X)δ(q,a,X) and δ(q,ϵ,X)\delta(q,\epsilon,X)δ(q,ϵ,X) cannot both nonempty
